Databases

In Memory


var database = [
	{
		id: 1,
		url: "http://google.ca/",
		shortcode: "3h532"
	},
	{
		id: 2,
		url: "http://lighthouselabs.ca",
		shortcode: "h43h3"
	}
];
  • In Memory databases are gone when the app restarts/crashes
  • What if you have more data than you do memory?

Flat File Style, with CSV

id,url,shortcode
1,http://google.ca/,3h532
2,http://lighthouselabs.ca/,h43h3
  • Flat files can serialize data
  • We can store a lot of it on disk
  • How would we access large data files that are bigger than our memory?

Structured style, with JSON

[
	{
		"id": 1,
		"url": "http://google.ca/",
		"shortcode": "3h532"
	},
	{
		"id": 2,
		"url": "http://lighthouselabs.ca",
		"shortcode": "h43h3"
	}
]
  • More structured, so easier for humans to read/edit
  • It is nearly like JavaScript, hence JSON == JavaScript Object Notation
  • Many more serialization formats appear for web and application development: BSON, gRPC, XML

CONS

  • Lots of overhead for the programmer
  • How do we find specific items?
  • How do we support searching while someone is updating?
  • What if I have lots of data?
- open, read/write, close. Sync'ing, locking, on and on and on. - Full scans - full locking or risk corruption - it may take a long time to do any CRUD operation

Appropriate Uses for flat-file

  • Static content/data
  • seed data for apps
  • Configuration
  • Meta data
- country/region mappings for UX - pre-populated data for ecomm, examples, etc - .env, settings.yml, etc - package.json, Gemfile, composer.json, etc.

MongoDB Intro

  • Stores data as arrays of objects (JSON style)
  • You interact with it with a Javascript query syntax
  • It takes care of the details behind: CRUD

Document storage

All your objects are now documents

All your documents are part of a collection

All your collections are in a database

- documents are rows - collections are tables - db's are... well... db's

Documents

  • dynamic schema
  • A few extra types supported: BSON
  • Can be embedded in other documents, or linked to other documents.
